# MPI による浅い部分木配布プロトタイプ

この文書は、リポジトリに追加した MPI プロトタイプの設計と使い方を簡潔にまとめたものです。

## 先に読むとよいもの

- [docs/bk_parallel_fundamentals.md](bk_parallel_fundamentals.md)
- [docs/openmp_taskization.md](openmp_taskization.md)

MPI は OpenMP より前提が重いので、まず共有メモリ並列との違いを押さえると入りやすいです。

目的:

- root (rank 0) が深さ `D` まで浅い部分木を生成して、それらをラウンドロビンで各 rank に配布する
- 各 rank は担当サブツリーをローカルで列挙し結果を標準出力に出す（将来的に集約する想定）

### このプロトタイプの位置づけ

これは「完成した分散 BK」ではなく、まず分散の最小構成を確認するための試作です。

修論では、この段階から次の問いを立てる必要があります。

- どういう分け方なら通信量が増えすぎないか
- どういう分け方なら負荷偏りが減るか
- OpenMP で見えた偏りを MPI に持ち込む価値があるか

使い方:

MPI 実行環境で次のように実行します:

```bash
mpirun -np 4 ./build/maximal_clique_mpi_bk graph.txt 2
```

注意:

- 現在の実装は単純化されたプロトタイプです。通信量や負荷均等化の改良が必要です。
- 実行には MPI C++ ライブラリが必要です（CMake で見つからない場合はビルドされません）。

### MPI で意識すべき基本概念

- プロセスはメモリを共有しない
- 送る情報は明示的にシリアライズする必要がある
- 通信回数が多いとすぐ遅くなる
- rank 間で仕事量が偏ると全体が遅くなる

### 修論での評価観点

- 何個のサブ問題に分割したか
- rank ごとの処理時間のばらつき
- 送受信したデータ量
- round-robin と重み付き配布の差
- OpenMP 版と比べてどこで有利・不利か

### 研究としての注意点

MPI は「2 台で動いた」だけでは弱いです。研究としては、分散方法の根拠と比較実験が必要です。

そのため、このプロトタイプの次にやるべきなのは、探索木ログからサブ問題の重さを推定して配布する方法を考えることです。
