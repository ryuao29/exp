# 再帰木解析ノートブックの説明

このページは、`notebooks/recursion_analysis.ipynb` の目的と使い方をまとめたものです。

## 目的

このノートブックは、Bron–Kerbosch の再帰木 CSV ログを対話的に解析し、探索木の偏りを見つけるためにあります。

このリポジトリの目標は複数台分散処理です。そのため、最初に必要なのは「どこが重いか」を定量的に把握することです。ノートブックはそのための観測装置です。

## できること

- CSV ログの読み込み
- 深さごとのノード数と `elapsed_us` の集計
- 部分木の `subtree_elapsed_us` の計算
- 重いノードの抽出
- 深さ分布と重い部分木の PNG 出力

## ここから何に使うか

このノートブックの結果を使って、次の閾値や分割規則を決めます。

- OpenMP の浅い深さ task 化条件
- MPI で配布する部分木の深さ上限
- ロードバランスの評価方法

具体的には、このノートブックで見つけた偏りを `docs/openmp_taskization.md` の task 深さ条件に反映します。

## 運用ルール

- BK のログ列や意味を変えたら、このノートブックを更新する。
- ノートブックの集計方法を変えたら、`docs/recursion_tree_logging.md` も更新する。
- コード変更が入ったら、説明書とノートブックの両方を更新する。

## 実行の目安

ノートブックは `matplotlib` と `pandas` を使います。通常は既に入っている想定ですが、無い場合は Python 環境に追加してください。

```bash
python3 -m pip install pandas matplotlib
```

ノートブックでは、既定で `/tmp/bk_log.csv` を読み込みます。別のログを使う場合は先頭のセルで `log_path` を変更してください。
