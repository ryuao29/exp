# Repository overview and How-To

This document summarizes what has been implemented so far and gives step-by-step instructions to understand, build, run, and extend the code and artifacts in this repository.

**Quick map**
- Code: `include/exp/` (common headers) and `src/` (executables)
- Tests: `tests/` (ctest)
- Docs: `docs/` (detailed notes), `notebooks/` (analysis notebook)
- Scripts: `scripts/` (analysis and run helpers)

## What's included (summary)

- Bitset: `include/exp/bitset.hpp` — word-based bitset with serialization helpers (`words()` / `set_words()`).
- Graph: `include/exp/graph.hpp` — graph loader and neighbor bitset generator.
- Logger: `include/exp/logger.hpp` & `include/exp/recursion_tree_logger.hpp` — CSV logger and recursion-tree logger.
- Bron–Kerbosch:
  - Sequential baseline: `include/exp/bron_kerbosch.hpp` + `src/maximal_clique_bk.cpp`.
  - OpenMP shallow-task prototype: `include/exp/bron_kerbosch.hpp` (task entry) + `src/maximal_clique_bk_omp.cpp`.
- Degeneracy ordering: `include/exp/degeneracy.hpp` + tests.
- MPI prototype: `src/maximal_clique_mpi_bk.cpp` (shallow-subtree generation at root and round-robin distribution). Conditional build if MPI found.
- Analysis tools:
  - `scripts/analyze_recursion_log.py` — CSV analysis and PNG plots (`/tmp/bk_analysis` by default).
  - `notebooks/recursion_analysis.ipynb` — interactive notebook to explore logs.
  - `scripts/run_mpi_prototype.sh` — helper to generate graph and run MPI prototype (prints instructions if MPI missing).

## Build

Prerequisites: C++17 compiler, CMake >= 3.16. Optional: OpenMP, MPI.

Build steps:

```bash
cmake -S . -B build
cmake --build build
```

- OpenMP targets are built when OpenMP is available. MPI targets are built only if an MPI C++ implementation is found by CMake.

## Tests

Run the test-suite with CTest:

```bash
ctest --test-dir build --output-on-failure
```

Key tests added:
- `bron_kerbosch_test` — correctness of sequential BK
- `degeneracy_test` — degeneracy ordering
- `bron_kerbosch_logging_test` — CSV logging presence
- `bron_kerbosch_openmp_test` — correctness of OpenMP prototype

## Running the sequential BK (baseline)

```bash
./build/maximal_clique_bk path/to/graph.txt
# optional recursion log:
./build/maximal_clique_bk path/to/graph.txt /tmp/bk_log.csv
```

Graph format: first line N (vertex count), following lines `u v` edges (0-indexed).

## Recursion log analysis workflow

1. Run BK with logging to produce `/tmp/bk_log.csv`.
2. Use the analysis script to summarize and plot:

```bash
python3 scripts/analyze_recursion_log.py /tmp/bk_log.csv --output-dir /tmp/bk_analysis
```

3. Or open `notebooks/recursion_analysis.ipynb` (it reads `/tmp/bk_log.csv` by default) and interactively inspect `depth`, `p_size`, and `subtree_elapsed_us` distributions.

Purpose: pick `task_depth` and `P_min` rules for taskization and distribution.

## Running the OpenMP prototype

```bash
./build/maximal_clique_bk_omp graph.txt TASK_DEPTH THREADS
```

Example: `./build/maximal_clique_bk_omp graph.txt 2 4` runs BK where nodes with depth < 2 spawn OpenMP tasks; uses 4 threads.

Note: The OpenMP version is a separate executable; sequential CLI remains unchanged.

## Running MPI prototype (on MPI-enabled host)

Build on an MPI-enabled machine:

```bash
cmake -S . -B build
cmake --build build --target maximal_clique_mpi_bk
```

Then run:

```bash
mpirun -np 4 ./build/maximal_clique_mpi_bk graph.txt DEPTH_LIMIT
```

Or locally generate and run with helper (prints build/run tips if MPI missing):

```bash
./scripts/run_mpi_prototype.sh --nodes 50 --prob 0.1 --depth 2 --procs 4
```

Remarks: Current MPI prototype does round-robin assignment of shallow subproblems generated by rank 0. It is a prototype and may need improvements (size-aware distribution, dynamic stealing).

## How to understand the code (step-by-step)

1. Read `docs/week0_foundation_modules.md` to get background rationale.
2. Inspect `include/exp/bitset.hpp` and `include/exp/graph.hpp` to see basic data structures.
3. Read `include/exp/bron_kerbosch.hpp` for the BK baseline and the task-parallel entrypoints (`enumerate_maximal_cliques_no_pivot` and `enumerate_maximal_cliques_no_pivot_task_parallel`).
4. Run the sequential BK on a small example (see tests) and compare outputs with the OpenMP executable to gain confidence.
5. Produce a recursion log and analyze it in `notebooks/recursion_analysis.ipynb` to pick task thresholds and to compute derived subtree statistics from `node_id`/`parent_id`.
6. Try OpenMP prototype with the chosen `TASK_DEPTH`. Measure speedup and imbalance.
7. If OpenMP shows promising splits, run MPI prototype on multiple machines and evaluate.

## Where to update docs when changing code

- If you change CSV columns or logger behavior: update `docs/recursion_tree_logging.md` and `tests/bron_kerbosch_logging_test.cpp`.
- If you modify BK APIs or CLI args: update `docs/openmp_taskization.md` and `README.md`.
- For any algorithmic change (degeneracy, pivoting): update corresponding docs in `docs/` and the notebook as needed.

## Next recommended steps
