# ドキュメント更新ルール (CONTRIBUTING_DOCS)

このファイルは、このリポジトリでドキュメントを更新・追加するときの最小限の運用ルールを示します。これに従うことで、コード変更とドキュメントの齟齬を防ぎます。

基本ルール:

1. コードの公開 API やログフォーマット、CLI オプションに変更を加えたら、必ず関連ドキュメントを更新すること。
2. ドキュメントは `docs/` 下に置く。小さな変更（ファイル内の1セクション追記）は既存ファイルの編集で良い。
3. 変更の理由と互換性への影響を短く書く（2-3 行で十分）。
4. テストがある場合は `tests/` を更新して、ドキュメントとの整合を CI で検査できるようにする。
5. コミットメッセージは `docs:` プレフィックスを付けること。

追加チェックリスト（PR 作成時に確認する）:

- 変更した API/CLI/ログの箇所を `docs/` に反映したか？
- 既存の説明書を破壊していないか？（リンク切れ、項目の矛盾）
- テストでドキュメント依存部分（ログヘッダなど）をカバーしているか？

採用例:

- `include/exp/recursion_tree_logger.hpp` に列を追加した場合:
  - `docs/recursion_tree_logging.md` を更新
  - `tests/bron_kerbosch_logging_test.cpp` の期待ヘッダに列名を追加
